# 블로그 프로젝트 설계 방향

## 1️⃣ 블로그의 목표

**누구나 쉽게 읽을 수 있고, 작성자가 편하게 관리할 수 있는 개인 블로그**

- ✅ 독자: 블로그에 방문해서 여러 글을 쉽게 찾아 읽기
- ✅ 작성자: 로그인 후 새 글을 작성하고, 작성한 글을 수정/삭제하기
- ✅ 모두: 깔끔하고 빠른 속도로 사용하기

---

## 2️⃣ 주요 사용자

### 👤 독자 (방문자)
- 블로그 링크를 통해 들어온 사람들
- **할 수 있는 것**: 글 목록 보기, 개별 글 읽기, 작성자 소개 보기
- **필요한 것**: 읽기 쉬운 글, 빠른 페이지 로딩

### ✍️ 작성자 (관리자)
- 블로그 소유자
- **할 수 있는 것**: 로그인, 글 작성, 글 수정, 글 삭제, 프로필 관리
- **필요한 것**: 글 작성 폼, 관리 대시보드, 간편한 인증

---

## 3️⃣ 꼭 필요한 페이지

| 페이지 | 설명 | 누가 봐? | 상태 |
|--------|------|---------|------|
| **홈 페이지** | 블로그 소개, 최근 글 미리보기 | 모두 | ✅ 완료 |
| **글 목록** | 작성된 모든 글을 보여줌 | 모두 | ✅ 완료 |
| **글 읽기** | 선택한 글의 전체 내용 보기 | 모두 | 🔄 개발중 |
| **글 작성** | 새 글을 작성하는 폼 | 작성자만 | ⏳ 예정 |
| **글 수정** | 작성한 글을 수정하는 폼 | 작성자만 | ⏳ 예정 |
| **마이페이지** | 작성자의 글 목록, 프로필 관리 | 작성자만 | ⏳ 예정 |

### 앞으로 추가될 수 있는 페이지
- 카테고리별 글 보기
- 검색 기능
- 댓글 기능

## 4️⃣ 디자인 분위기

### 📐 디자인 원칙: **심플하고 깔끔하게**

**색상**
- 주색: Shadcn/ui의 기본 파란색 (신뢰감, 전문성)
- 배경: 하얀색 (깔끔함, 글 읽기 편함)
- 텍스트: 검정색/진회색 (높은 가독성)

**구성**
- 제목은 크고 명확하게
- 글들은 카드 형태로 구분 (그림자 효과)
- 버튼과 입력창은 모서리가 둥근 모양
- 여백(공간)을 충분히 사용해서 답답하지 않게

**반응형 디자인**
- 휴대폰: 1열 (가로 전체 사용)
- 태블릿/PC: 필요에 따라 2열 이상
- 모든 화면 크기에서 읽기 편해야 함

**예시**
```
┌─────────────────────────────────┐
│         블로그 제목             │
│    글쓰기 | 로그인              │
├─────────────────────────────────┤
│  ┌──────────┐ ┌──────────┐     │
│  │ 글제목 1 │ │ 글제목 2 │     │
│  │ 요약...  │ │ 요약...  │     │
│  └──────────┘ └──────────┘     │
└─────────────────────────────────┘
```

## 5️⃣ Copilot에게 코드 요청할 때 지켜야 할 기준

### ✅ **해야 할 것**

1. **shadcn/ui 컴포넌트 사용**
   - Button, Card, Input, Dialog 등 미리 만들어진 컴포넌트 사용
   - `components/ui/` 폴더에서 찾기

2. **Tailwind CSS로 스타일링**
   - 색상은 반드시 디자인 토큰(CSS 변수) 사용: `bg-[--background]`, `text-[--foreground]`
   - Tailwind 클래스만 사용 (직접 CSS 쓰지 말기)
   - 예: `className="space-y-6 p-6 rounded-lg"`

3. **Server Component 기본값**
   - 필요 없으면 `"use client"` 쓰지 말기
   - 상호작용 필요할 때만 Client Component 사용

4. **폴더 구조 지키기**
   - 페이지: `app/` 폴더
   - UI 컴포넌트: `components/ui/`
   - 기타 컴포넌트: `components/`
   - 유틸리티 함수: `lib/`

5. **파일명 규칙**
   - 폴더명: 소문자, 하이픈 사용 (`my-page`, `post-list`)
   - 파일명: 파스칼 케이스 (`PostCard.tsx`, `Header.tsx`)

### ❌ **하면 안 되는 것**

1. **pages/ 라우터 사용** → App Router 사용하기
2. **next/router 사용** → next/navigation 사용하기
3. **직접 색상값 사용** → CSS 변수 사용하기
   - ❌ `bg-blue-500`
   - ✅ `bg-[--primary]`
4. **불필요한 Client Component**
   - 서버에서 처리할 수 있으면 Server Component 사용
5. **외부 라이브러리 임의로 추가** → 기존 라이브러리 활용 먼저 확인

### 📋 **코드 요청 양식 (예시)**

```
"포스트 목록을 보여주는 컴포넌트를 만들어줘.
- 포스트는 카드 형태로 보여주기
- 제목, 날짜, 요약 3줄 포함
- 모바일에서는 1열, PC에서는 2열 그리드
- shadcn/ui Card와 Button 사용"
```
