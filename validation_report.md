## 3. 데이터베이스(DB) 스키마 관련 잠재 이슈 및 개선 사항

프로젝트 품질 향상 및 안정적인 컴포넌트 렌더링을 위해 진단된 데이터베이스 구조적 개선 포인트는 다음과 같습니다.

1. **`posts` 테이블 내 `author_email` 누락 현상 수정**
   - 현재 `posts` 테이블에 `author_email` 컬럼이 설계되어 있지 않아, 상세 화면 진입 시 작성자 정보가 항상 고정된 하드코딩 텍스트인 **"익명 작성자"**로 노출되는 UI 버그가 식별되었습니다. 
   - 사용자의 실제 메일 주소를 바인딩할 수 있도록 `posts` 테이블에 `author_email (TEXT)` 필드를 추가하는 스키마 마이그레이션이 권장됩니다.

2. **`comments` 및 `likes` 테이블 마이그레이션/RLS 파일 누락 대응**
   - Supabase 원격 환경 구축 과정에서 `comments` 테이블의 ID 데이터 타입 구조 충돌(`bigint` vs `uuid`) 및 Row Level Security(행 레벨 보안) 조회 권한 누락으로 인해, 프론트엔드가 댓글 목록을 불러오지 못하고 하단 전체 레이아웃이 굳어버리는(Render Lock) 치명적인 예외가 진단되었습니다.
   - 이를 방지하기 위해 Next.js 코드 내부에 `(comments || []).length` 와 같이 철저한 널 가드(Null Guard) 예외 방어 처리를 반영하여 페이지 크래시를 원천 방어했으며, 향후 재현성 확보를 위해 스키마 `DDL/RLS` 설정 마이그레이션 SQL 스크립트 파일을 프로젝트 루트 내에 명시적으로 추가 보관하는 구조적 보강을 제안합니다.
