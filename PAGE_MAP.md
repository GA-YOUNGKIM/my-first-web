# 페이지 맵과 유저 플로우 — my-first-web

개인 블로그의 모든 페이지를 한눈에 보고, 사용자가 어떻게 움직이는지 이해하기 위한 문서입니다.

---

## 1. 전체 페이지 맵

### 표로 정리한 페이지 구조

| 페이지 | URL | 파일 경로 | 설명 | 누가 볼 수 있나 |
|---|---|---|---|---|
| **홈** | `/` | `app/page.tsx` | 블로그 소개, 주요 진입 지점 | 누구나 |
| **글 목록** | `/posts` | `app/posts/page.tsx` | 모든 글을 한눈에 보기 | 누구나 |
| **글 상세** | `/posts/[id]` | `app/posts/[id]/page.tsx` | 한 글을 크게 읽기, 수정/삭제 | 누구나 (수정/삭제는 작성자만) |
| **글 작성** | `/posts/new` | `app/posts/new/page.tsx` | 새 글 작성하기 | **로그인한 사용자만** |
| **글 수정** | `/posts/[id]/edit` | `app/posts/[id]/edit/page.tsx` | 이미 쓴 글 고쳐 쓰기 | **작성자만** |
| **로그인** | `/login` | `app/login/page.tsx` | 이메일/비밀번호로 로그인 | 로그인하지 않은 사람만 |
| **회원가입** | `/signup` | `app/signup/page.tsx` | 새 계정 만들기 | 로그인하지 않은 사람만 |
| **마이페이지** | `/mypage` | `app/mypage/page.tsx` | 내 프로필, 내가 쓴 글 모음 | **로그인한 사용자만** |

### 아이콘 설명
- 📖 = 읽기만 (누구나 볼 수 있음)
- ✍️ = 쓰기 필요 (로그인 필수)
- 🔒 = 제한 있음 (작성자, 본인만)

---

## 2. 유저 플로우 (사용자가 움직이는 흐름)

### 🧑‍💻 플로우 1: 글 읽기 (독자 입장)

```
1. 홈 (/)
   ↓
2. "글 목록" 클릭 → 글 목록 (/posts)
   ↓
3. 글 카드 클릭 → 글 상세 (/posts/1)
   ↓
4. 글 읽기 완료
```

**간단히:** 홈 → 글 목록 → 글 상세

---

### ✍️ 플로우 2: 글 작성 (작성자 입장)

```
1. 홈 (/)
   ↓
2. "로그인" 클릭 → 로그인 (/login)
   ↓
3. 이메일/비밀번호 입력 → 로그인 성공
   ↓
4. 다시 홈으로 (자동 리다이렉트 또는 수동)
   ↓
5. "새 글 쓰기" 버튼 클릭 → 글 작성 (/posts/new)
   ↓
6. 제목, 내용 입력
   ↓
7. "저장" 버튼 → 글 상세 (/posts/새글ID)
   ↓
8. 글 작성 완료
```

**간단히:** 홈 → 로그인 → 글 작성 → 글 상세

---

### 👤 플로우 3: 마이페이지 확인 (작성자 입장)

```
1. 어디서든 상단 메뉴의 "마이페이지" 클릭 → 마이페이지 (/mypage)
   ↓
2. 내 프로필 확인
   - 이름, 이메일, 가입일 등
   ↓
3. 내가 쓴 글 목록 확인
   ↓
4. 글 클릭 → 글 상세 (/posts/ID)
   ↓
5. "수정" 또는 "삭제" 버튼으로 관리
   ↓
6. 수정하면 → 글 수정 (/posts/ID/edit)
   ↓
7. 변경사항 저장 → 글 상세로 돌아옴
```

**간단히:** 마이페이지 → 내 글 확인 → 수정/삭제

---

### 💬 플로우 4: 로그아웃 (작성자 입장)

```
1. 상단 메뉴에서 "로그아웃" 버튼 클릭
   ↓
2. 세션 삭제
   ↓
3. 홈으로 자동 리다이렉트 (또는 로그인 페이지)
   ↓
4. 로그아웃 완료
```

---

## 3. 사용자 역할별 볼 수 있는 페이지

### 👤 손님 (로그인 안 한 사람)

```
✅ 볼 수 있음:
  - 홈 (/)
  - 글 목록 (/posts)
  - 글 상세 (/posts/[id])
  - 로그인 (/login)
  - 회원가입 (/signup)

❌ 볼 수 없음:
  - 글 작성 (/posts/new) → 로그인 페이지로 이동
  - 글 수정 (/posts/[id]/edit) → 로그인 페이지로 이동
  - 마이페이지 (/mypage) → 로그인 페이지로 이동
```

### 🔓 로그인한 사용자

```
✅ 볼 수 있음:
  - 모든 페이지
  - 특히 글 작성, 글 수정(자신의 글), 마이페이지
  - 로그아웃 버튼

❌ 볼 수 없음:
  - 다른 사람의 글 수정/삭제 (버튼이 안 보임)
  - 로그인/회원가입 페이지 (이미 로그인했으므로)
```

## 4. 페이지별 주요 액션 (버튼, 링크)

### 홈 (/)
- 글 목록 보기 → `/posts`로 이동
- 새 글 쓰기 → `/posts/new`로 이동 (로그인 필수)
- 로그인 → `/login`으로 이동

### 글 목록 (/posts)
- 글 클릭 → `/posts/[id]`로 이동
- 새 글 쓰기 → `/posts/new`로 이동 (로그인 필수)

### 글 상세 (/posts/[id])
- 목록으로 돌아가기 → `/posts`로 이동
- 수정 버튼 → `/posts/[id]/edit`로 이동 (작성자만 보임)
- 삭제 버튼 → 확인 후 삭제, `/posts`로 이동 (작성자만 보임)

### 글 작성 (/posts/new)
- 제목 입력, 내용 입력
- 저장 버튼 → 글 작성 후 `/posts/[새글ID]`로 이동
- 취소 버튼 → `/posts`로 이동

### 글 수정 (/posts/[id]/edit)
- 기존 내용 불러오기
- 수정 후 저장 → `/posts/[id]`로 이동
- 취소 버튼 → `/posts/[id]`로 이동

### 로그인 (/login)
- 이메일 입력, 비밀번호 입력
- 로그인 버튼 → 성공 시 `/posts`로 이동
- 회원가입 링크 → `/signup`로 이동

### 회원가입 (/signup)
- 이메일 입력, 비밀번호 입력
- 회원가입 버튼 → 성공 시 `/login`으로 이동 (이메일 인증 후)
- 로그인 링크 → `/login`으로 이동

### 마이페이지 (/mypage)
- 프로필 확인
- 내 글 목록 확인
- 글 클릭 → `/posts/[id]`로 이동
- 로그아웃 버튼 → 로그아웃 후 `/` 또는 `/login`으로 이동

## 5. 빠진 페이지 제안

### 제안 1: 404 Not Found 페이지 (`/404`)
**이유:** 없는 글(`/posts/999`)을 요청했을 때 보여줄 페이지
**필요성:** ⭐⭐⭐ (중요)
**구현:** `app/not-found.tsx`

```
홈으로 돌아가기 버튼 제공
```

---

### 제안 2: 검색 페이지 (`/search`)
**이유:** 글이 많아지면 특정 글을 빠르게 찾아야 함
**필요성:** ⭐⭐ (중요도 중)
**예시:** `/search?q=react` → "react" 포함된 글만 보기

```
검색창 입력 → 결과 표시
```

---

### 제안 3: 마이페이지 서브 페이지 (`/mypage/edit`)
**이유:** 프로필 수정 (이름, 프로필 사진 등)
**필요성:** ⭐⭐ (중요도 중)

```
이름 수정, 소개 수정, 비밀번호 변경 등
```

---

### 제안 4: 댓글 기능 (`/posts/[id]#comments`)
**이유:** 글에 대한 독자 반응
**필요성:** ⭐⭐⭐ (중요, 나중 단계)
**구현:** `/posts/[id]` 페이지에 댓글 섹션 추가

---

### 제안 5: 카테고리 페이지 (`/category/[name]`)
**이유:** 글을 주제별로 분류
**필요성:** ⭐ (선택사항)
**예시:** `/category/react` → React 관련 글만 보기
